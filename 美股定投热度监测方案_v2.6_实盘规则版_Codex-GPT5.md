# 美股定投热度监测系统设计方案(v2.6 · 实盘规则版)

> 模型名称: OpenAI Codex (GPT-5)
> 文件名: `美股定投热度监测方案_v2.6_实盘规则版_Codex-GPT5.md`
> 修订日期: 2026-06-13
>
> v2.6 结论: v2.5 的指标体系基本保留, 但执行层重写。系统不再用 Heat 直接决定每月投 0x/0.5x/1.5x/2x; 核心月度 1x 定投永远继续。Heat 与估值/趋势只用于少数高估破位阶段的风险现金状态机。
>
> 回测依据: `backtest_heat_v26.py` 与 `backtest_outputs/v26_backtest_report.md`。使用截至约 2026-06-11 的本地数据, 2007-01 起完整覆盖 SPX/NDX、CAPE、Buffett、ECY、VIX、Put/Call、HY OAS 与 SPX 200DMA 广度。
>
> 实盘边界: 本方案是规则化执行框架, 不是收益保证。若后续滚动回测显示 SPX 或 NDX 任一侧重新落后固定 1x 定投, v2.6 自动降级为观察规则, 不继续作为实盘主规则。

---

## 1. 为什么废弃 v2.5 执行表

v2.5 的主要问题不在数据接入, 而在执行层过度择时:

- `60-75` 偏热档降到 0.5x, `75-88` 过热档暂停定投, 导致长期牛市阶段少买太多。
- 蓝档/冷档出现次数很少, 后续加倍买入不足以补回前期少买。
- 2020 年 3 月恐慌标记触发, 但慢变量估值仍偏高, 总 Heat 没有落入极度冰点, 说明单一 Heat 不适合同时承担买入和风控。
- v2.5 月度模拟相对固定定投落后约 8%, 最大回撤改善很小, 不适合直接实盘。

v2.6 因此只做一件事: 把系统从"定投金额调节器"改为"固定定投 + 少数风险现金切换"。

---

## 2. v2.6 核心原则

1. **核心定投不停。** 每月第一个交易日投入 1 单位, 默认全部买入目标资产。系统不再因为偏热而停止普通定投。
2. **风险现金只在双重条件下启用。** 只有估值处于历史高位且趋势已经破位, 才允许把一小部分已有仓位转入现金/短债。
3. **先验参数优先。** 使用标准 200 日均线、估值前 10%、连续 10 个交易日确认、20% 风险现金上限; 不做大规模网格搜索。
4. **低频可执行。** 所有触发都基于收盘后数据, 次一交易日执行。没有日内判断, 没有杠杆, 没有期权。
5. **现金有收益。** 风险现金默认放货币基金、短债 ETF 或同等短期国债工具; 回测用 H.15 3-month Treasury constant maturity 作为现金收益代理。

---

## 3. 继续保留的指标体系

v2.6 沿用 v2.5 的指标计算与防前视规则:

- 估值: CAPE、ECY、Buffett
- 情绪: VIX、VIX 期限结构、Put/Call
- 趋势: 200DMA 偏离、回撤、周线 RSI
- 广度/信用: SPX 200DMA 广度、HY OAS
- 修正项: SPX/NDX 分别计算热度, 保留 NDX 相对过热、广度背离与恐慌标记

但 v2.6 不再把 `Heat` 当作单一买卖分数。`Heat` 只参与两个地方:

- `Heat < 35` 视为市场已经进入冷区, 可触发风险现金回补。
- `VIX 分位 >95 且 VIX > VIX3M` 视为恐慌贴水, 可触发风险现金回补。

---

## 4. 实盘执行规则

### 4.1 每月核心定投

每月第一个交易日:

```
买入目标资产 1x
```

这一步永不因为 Heat 偏热而取消。若账户同时定投 SPX 与 NDX, 两个资产各自执行自己的 1x。

### 4.2 风险现金触发

对每个资产袖珍组合分别判断。SPX 用 SPX 价格与 200DMA; NDX 用 NDX 价格与 200DMA。估值分位使用同一套市场估值分位。

当以下条件连续满足 **10 个交易日** 时, 次一交易日进入风险现金状态:

```
CAPE 分位 >= 90
且 Buffett 分位 >= 90
且 (价格 < 200DMA 或 自历史高点回撤 > 10%)
且 不处于冷区/恐慌回补状态
```

进入风险现金状态时:

```
卖出该资产袖珍组合的一部分, 使现金/短债约等于组合市值的 20%
```

注意: 这是一次性风险降档, 不是持续动态再平衡。进入风险状态后, 后续每月 1x 核心定投仍继续买入目标资产, 不额外卖出新买入部分。

### 4.3 风险现金回补

处于风险现金状态时, 若以下任一条件成立, 次一交易日把风险现金全部买回目标资产:

```
Heat < 35
或 VIX 分位 > 95 且 VIX > VIX3M
或 CAPE 分位 < 90
或 Buffett 分位 < 90
或 价格连续 10 个交易日 > 200DMA
```

回补后恢复普通状态:

```
每月 1x 定投, 无风险现金
```

### 4.4 同时持有 SPX 与 NDX 时

SPX 与 NDX 的风险现金状态独立。可能出现:

- SPX 普通, NDX 风险现金
- SPX 风险现金, NDX 普通
- 两者同时风险现金

每个资产袖珍组合的现金上限都是该袖珍组合自身的 20%, 不跨资产挪用。

## 6. 反过拟合约束

v2.6 明确禁止以下做法:

- 不按年度收益回头调阈值。
- 不为了捕捉 2020 或 2021 单独增加特殊案例。
- 不把 150DMA、30% 风险现金这类回测更高的组合写成主规则。
- 不把 `Heat` 重新拟合成买卖回归模型。
- 不扩大减仓上限; 风险现金上限固定为 20%。

每次更新数据后, 必须重跑:

```
python backtest_heat_v26.py
```

通过条件:

```
SPX strategy_vs_fixed_pct > 0
且 NDX strategy_vs_fixed_pct > 0
且 max_cash_pct <= 25
```

若任一条件不满足, v2.6 不再作为实盘主规则。

## 7. 实盘执行清单

每日收盘后:

1. 更新价格、CAPE、Buffett、VIX/VIX3M、Heat。
2. 对 SPX 与 NDX 分别更新风险状态计数器。
3. 若风险触发计数达到 10 天, 次一交易日把对应资产袖珍组合调到约 20% 现金/短债。
4. 若处于风险现金状态且回补条件触发, 次一交易日全部买回。

每月第一个交易日:

1. SPX 定投 1x。
2. NDX 定投 1x。
3. 即使处于风险现金状态, 这 1x 也继续买入。

现金工具:

- 优先使用货币基金、短债基金、短期国债 ETF 或券商现金管理工具。
- 不使用长债替代风险现金; 长债有久期风险, 不等同现金。
- 应避开会影响生活备用金的资金。风险现金只是投资组合内部的战术现金, 不是生活现金。

---

## 8. 剩余风险

- 当前完整回测从 2007 年开始, 大顶样本仍然少。结果通过, 但不能视为统计定律。
- 最大回撤没有明显改善; v2.6 的价值是略微提高终值并降低高估破位期的心理压力, 不是防崩盘保险。
- 回测未计税费。事件数低, 但应优先在免税/递延账户或低成本 ETF 中执行。
- Yahoo 指数价格不是 ETF 总回报序列。相对固定定投的比较仍有参考价值, 但真实 ETF 会受分红、费用、滑点影响。
- 未来若美国市场长期估值结构改变, CAPE/Buffett 的历史 90 分位阈值可能失效, 必须滚动复核。
